業界問題

我們鎖定「中型/小型公司」會計作業最耗時、最容易錯的環節。

多數中小企業在會計流程上面臨的是「大量重複 + 高風險」的工作：

  • 交易分類與分錄：依賴人工判斷，學習成本高、錯誤難即時發現。
  • 銀行調節/對帳：反覆比對明細、找差異、補分錄，是最耗時間的核心痛點。
  • 財報產出與修正：月底/季底才發現錯，常需要回頭補帳，成本更高。
  • 缺乏追溯能力：報表數字找不到明確來源，影響內控、稽核與決策。

我們的結論：真正浪費人力的不是「輸出報表」而是「抓差異、調節、修正、重做」。

專案必要性

現有會計軟體通常提供「輸入/查詢/輸出」的工具，但在中小企業實務中， 最痛的是 對帳與調節、以及 大量交易的分類與分錄建置。

LLM 的價值不在於取代會計判斷，而是把「判斷」變成「可審核的建議」： 讓會計人員從「從零開始做」變成「快速確認與修正」。

因此，本專案要做的是一個能落地的 Prototype： LLM 提案 → 人類核准 → 調節表兜底 → 財報可追溯， 把最花時間、最容易錯的環節先解決。

解決方案

我們要做的是「完整閉環」，不是只做報表產生器。

核心設計原則：

  • 資料庫優先：交易/分錄/調節/輸出歷程全入庫，才能追溯與重現。
  • LLM 為主：輸出分錄草案 + 理由 + 信心（人可以審核）。
  • 調節表為核心：銀行調節與 AR/AP 對帳，將差異變成任務清單。
  • 檢核兜底：平衡檢查、合理性規則、期間一致性，降低錯誤率。

你在台上可以用一句話收斂：「我們不是要取代會計，是要縮短會計工作中的重複與抓錯時間。」

產品功能

每個模組都能獨立交付，也能組成完整流程。

LLM會計助手（分類/分錄草案）

讀取交易摘要、對象、用途描述，提出科目與分錄草案，並提供理由與信心分數， 讓會計人員用「審核」取代「從零開始」。

調節表/對帳模組（核心痛點）

銀行調節、AR/AP 對帳：自動比對明細、標出差異、生成待辦清單， 並建議可能的調整方向（例如漏帳、重覆入帳、期間錯置）。

財報生成（報表 + 明細）

產生損益表、資產負債表（先做兩張最穩），並提供 drill-down 明細， 讓每個數字都能追到分錄與交易。

檢核與風控

平衡檢查、合理性規則、期間一致性、異常跳動提示。 目的不是「限制」，而是把錯誤在輸出前抓掉，降低補帳成本。

系統成本

成本結論：專題階段不需要募資，成本可控，重點在「取得企業訪談與案例回饋」。

  • LLM API：依使用量計費（可設定上限），專題 demo 規模成本低。
  • 伺服器/部署：可用低成本雲端或校內/個人設備；demo 不吃重規模。
  • 資料庫：採開源（例如 PostgreSQL / SQLite）即可。

預期改善

  • 會計處理時間：透過「分錄草案 + 調節清單」降低重複作業（目標：-40%）。
  • 錯誤率：透過檢核兜底與可追溯流程，降低月底補帳（目標：-60%）。
  • 報表產出效率：把大量時間從「整理」轉為「確認」（目標：速度提升數倍）。
  • 內控/稽核友善：每個數字可追到交易與分錄來源，降低溝通成本。

注意：這些是「專題目標指標」，最後可用 demo 案例呈現達成程度。

流程詳細說明

每步驟都包含：目的、輸入、輸出、我們要做的重點。

1｜資料匯入

目的：把企業現有資料先導入，建立可重跑、可驗證的資料來源。

輸入：交易明細（Excel/CSV）、銀行對帳資料（可選）、憑證清單（可選）。

輸出：原始資料版本（raw）＋ 匯入紀錄（時間/來源/欄位對應）。

重點：MVP 不追求「所有系統都能接」，先做到「可匯入、可重現、可追溯」。

2｜資料庫入庫（標準化）

目的：沒有一致資料模型，LLM/調節/報表都會不穩。

輸入：raw 資料。

輸出：標準化表（transactions / journal / reconciliation / outputs）。

重點：加上基本索引/查詢路徑，讓後續搜尋與對帳效率穩定。（你提到的資料結構最佳化點到即可）

3｜LLM會計助手（分錄草案）

目的：把「人工判斷」變成「可審核的建議」，縮短作業時間。

輸入：交易摘要、對象、用途、金額、日期（必要欄位）。

輸出：分錄草案（借貸/科目）＋ 理由（文字）＋ 信心分數（0~1）＋ 缺件提醒。

重點：LLM 不能直接「過帳」，一定要經過人工審核；我們做的是「建議＋解釋」不是「自動做帳」。

4｜調節表/對帳（核心亮點）

目的：會計最耗時的是抓差異。這步驟把差異變成待辦清單。

輸入：帳上交易/分錄 + 銀行對帳資料 / AR/AP 明細（擇一或擇二）。

輸出：調節表（reconciliation schedule）＋ 差異清單（每筆差異的推測原因）＋ 建議調整方向。

重點：MVP 建議先做 2 種：銀行存款調節 + AR/AP 對帳，就已經很有業界價值。

5｜人工審核（人機協作落地點）

目的：確保穩定性與責任歸屬：LLM 提案、人類核准。

輸入：分錄草案 + 調節差異清單。

輸出：核准分錄、修改紀錄、補件狀態（例：待憑證）。

重點：這一步也是防弊：任何變更都留下紀錄，後續可追溯。

6｜報表生成 + 檢核（降低錯誤率）

目的：快速產出可用報表，並在輸出前做風控檢核。

輸入：核准後分錄 + 帳務彙總。

輸出：損益表、資產負債表（MVP）＋ 明細 drill-down ＋ 檢核結果（平衡/合理性/期間一致性）。

重點：現金流量表可作為後續擴充；先把「兩張最穩」做完整並可追溯。

7｜輸出交付（投資案式成果包）

目的：讓成果可交付、可展示、可驗證。

輸出內容：報表（PDF/Excel）＋ 報表明細 ＋ 調節表 ＋ 問題清單 ＋ 作業歷程。

重點：期末展示時，你可以用「一個月的案例」從匯入到輸出完整跑一遍，最有說服力。
